how a decision happens the room comes before the number

Two funnels, one screen: inbound applications and outbound-discovered founders converge into the same pipeline — application + public signals + an explicit fund thesis in, a priced decision out. The load-bearing inversion: a forum of real named voices is built first, and the score is a projection of that room, computed last. Scores never enter the room.

the structure three layers · one contract
the honesty contract
  • evidenceEvery claim traces to evidence at a stated confidence tier; gaps are flagged, never silently filled.
  • axesThe three screening axes are never collapsed into one number — when they disagree, that disagreement is the finding.
  • scoreThe Founder Score is one input to the founder axis, not a substitute for it.
  • cold startA founder with no funding history, no network and a thin footprint is priced transparently, not penalised by default.
  • humanOne human holds the gate. The system drafts the decision; it does not wire the money.

the distance to the terminal weeks of engineering · one honest long game

The destination is a terminal: live feeds in, analysis running whether or not anyone is watching, and a login that shows the current state of every founder we track. This page is not that yet — it is a static build over committed run tapes, and it says so. But the distance is shorter than it looks, because the always-on parts already exist as batch machinery. What follows is the exact inventory: built, not yet, and the path between.

  • builtThe inverted pipeline with a standing counsel — screen → triage → forum → counsel → memo, an explicit fund thesis injected at every stage. ferrite-inverted is the first tape to come back green from every artifact verifier.
  • builtA measured sourcing gate — 580 real archived repos pushed through the harvest → junk/thin/pass gate, where “pass” is defined as opening the pipeline's input gate, so sourcing and pipeline cannot drift apart.
  • builtA batch farm runner — the full pipeline over any folder of sourced applications, on three interchangeable workers (API, cursor-agent, headless Claude); a 25-run person-seeded overnight trial is on disk. “Analysis all the time” is a scheduler entry away, not an invention away.
  • builtSpeed as evidence — a real outbound candidate went from public GitHub signals to a decision-ready memo in 794 seconds, with a mechanical (code, not prompt) forum fire gate; the fastest run on record closed in 171 seconds (real-candidate tapes stay private pending consent).
  • not yetA live store — memory is JSON files in git; the persistent store that everything reads and writes is in progress. Until it lands, “dynamic update” means “rebuild and recommit”.
  • not yetScheduled harvesters — sourcing has only ever run on demand against archives. No live collectors, no watermarks, no delta detection. The gate is proven; the cadence is not.
  • not yetA served terminal — no server, no accounts, no live queries. The viewer renders what was true at the last corpus build.
  • the pathStore → harvesters → triggers → serving, in that order, each unblocking the next: the store becomes the single read/write path (tapes become views of it); collectors write deltas through the existing gate on a schedule; a tiered ladder reacts to deltas — deterministic score always, triage on threshold, a full room only on material change; then a thin authenticated API and this viewer reads live, with watchlists and alerts. Each step is days-to-a-week of agent-driven engineering on machinery that already runs in batch.
  • the long gameBloomberg-grade validated feeds — entity resolution across sources, licensed data, and validation ops are the one part that does not compress into weeks. We say so instead of pretending: every signal carries provenance from day one so that moat can be built without rework.

deliberate cuts — hosted DBs, submission platforms, breadth-over-depth channels — are cuts, not gaps. full map: docs/ROADMAP.md
